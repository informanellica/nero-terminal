On this page

セッションマネージャ — メインプロセスにおける nero-terminal の PuTTY 風セッション構成とライフサイクルを担います。

リモートの SSH セッション（SshHost ／ ssh2）またはローカルシェル（PtyHost ／ node-pty）のいずれかを開きます。両者は同じホストインターフェースを実装するため、同一のターミナルライブラリ IPC ブリッジ（attachPtyToIpc）がどちらのバックエンドにも対応します。また、ローカルシェルの検出や「Saved Sessions」の userData/sessions.json への永続化も行います。

scripts/session-smoke.js によってヘッドレスで実行できるよう、module:main/main から切り出されています。

Methods

(inner) buildJumpList(execPath, profiles, iconPathopt) → {Array.<object>}

保存済みセッションプロファイルから Windows タスクバーのジャンプリストを構築します（app.setJumpList() 用）。各エントリはアプリを --open-profile="<name>" で再起動し、起動中のインスタンスがその保存済みセッションを開きます。プロファイルが無い場合は空配列を返します（リストをクリアします）。

profilesArray.<SessionProfile>

一覧表示する保存済みプロファイル。

各ジャンプリストエントリに表示するアイコン。

app.setJumpList() 用のカテゴリ配列。

(inner) detectShells() → {Array.<ShellDescriptor>}

このマシン上で利用可能なローカルシェルを検出します（「Local」接続方式向け）。Windows の場合: PowerShell、pwsh 7（存在すれば）、コマンドプロンプト、WSL（存在すれば）、Git Bash（存在すれば）。それ以外の環境では $SHELL に加えて一般的な代替シェル。
