On this page

nero-terminal の Tera Term 風ウィンドウのレンダラーです。

アーキテクチャ:

  • 起動時に生成され常に表示される、単一の永続的な xterm ターミナル（module:renderer/renderer~ensureTerminal）。これにより常に正しく描画されます（非表示／再表示の切り替えは行いません — これは以前 xterm の初回描画を壊す原因でした）。
  • セッションの構成は、ターミナル上に重ねて表示するオーバーレイダイアログで、起動時に表示され、タイトルバーの歯車ボタンから再度開けます。
  • module:renderer/renderer~startSession は window.sessionAPI を通じてセッションを開き、その後 window.terminalAPI のバイトをビューに流し込みます。
  • UI テキストは I18n（data-i18n 属性＋ t()）でローカライズされ、ライト／ダークテーマには Bootstrap の data-bs-theme を使用します。

ターミナルバックエンド（PTY／SSH）と xterm ラッパーは nero_modules/ 配下の共有ライブラリから提供されます。

Methods

(inner) applyLiveTerminalOptions()

現在の表示設定（フォント・カーソル・スクロールバック・配色テーマ）を、セッションを開き直さずに稼働中の永続ターミナルへ即時適用します。フォント拡大などの変更がセッション中でもすぐに反映されます。

(inner) closeSettings()

構成オーバーレイを非表示にし、フォーカスをターミナルに戻します。

(inner) ensureTerminal()

単一の永続的な TerminalView（xterm ＋ fit ＋ canvas レンダラー）を一度だけ生成し、#terminal にマウントします。冪等です。ターミナルは表示された状態で生成され、決して非表示にされないため、xterm は常に正しく描画されます。

(inner) flash(msg, isError)

ダイアログのフッターに一時的なメッセージを表示します。

メッセージテキスト（空でクリア）。

(inner) formToProfile() → {object}

構成フォームを SessionProfile の形のオブジェクトとしてスナップショットします。保存とセッションのオープンの両方で使用されます。

(async, inner) init() → {Promise.<void>}

レンダラーをブートストラップします: 静的な DOM をローカライズして言語ピッカーを接続し、永続的なターミナルを生成し、すべてのイベントハンドラを同期的にアタッチし（非同期セットアップが遅かったり失敗したりしてもボタンが機能するように）、カラーテーマとインストール済みフォントのピッカーを構築し、設定オーバーレイを表示してから、検出されたシェルと保存済みセッションを読み込みます。

Returns:
Type: 
Promise.<void>

(async, inner) openProfileByName(name) → {Promise.<void>}

保存済みセッションプロファイルを名前で読み込み、即座に開始します。Windows のジャンプリスト項目（または --open-profile 起動引数）で保存済みセッションが選択されたときに呼ばれます。プロファイルが見つからない場合は設定ダイアログを表示します。

Type: 
Promise.<void>

(inner) openSettings()

セッション構成オーバーレイをターミナル上に表示し、ホストフィールドにフォーカスします。

(inner) resolveLaunch(p) → {object}

フォームのスナップショットを window.sessionAPI.open 用のペイロードに変換し、あわせてタイトルバー用の短い表示用 label も生成します。必須フィールド（SSH ホスト、またはカスタムコマンド／シェル）が欠けている場合は、ローカライズされた Error をスローします。

Bootstrap のライト／ダーク UI テーマを適用し、月／太陽アイコンを切り替え、それに合わせてネイティブタイトルバーのオーバーレイを配色し直すようメインプロセスに依頼します。

(async, inner) startDefaultSession() → {Promise.<void>}

手動設定を省略して、デフォルトのローカルシェルを即座に開きます。「起動時にデフォルト端末を開く」トグルの動作を担い、要求されたプロファイルを開けない場合の妥当なフォールバックにもなります。

Returns:
Type: 
Promise.<void>

(async, inner) startSession() → {Promise.<void>}

現在のフォーム値でセッションを開き、永続的なターミナルに接続します。以前のセッションがあれば破棄し、セッションごとの xterm オプション（フォント／テーマ／スクロールバック／カーソル）を適用して画面をリセットし、ビューの準備が整うまで初期の出力をバッファリングしてから、terminalAPI ⇄ ビューを接続してオーバーレイを閉じます。バリデーションや接続のエラーは flash を介してインライン表示されます。

Returns:
Type: 
Promise.<void>
